TÂCHE 2 — MODULE RÉUNIONS (MEETINGS)
================================================================================

## Objectif
Module complet : nom, date/heure, organisateur (famille/groupe/organisation), présences
présent/absent, remarques, menu navigation.

## N'existait pas
Aucun module meeting ; le module Events (events_event) est différent (check-in calendrier).

## Organisateur — modèle de données
Table meeting_mtg :
- mtg_Name, mtg_DateTime, mtg_OrganizerType, mtg_OrganizerId, mtg_Remarks
- mtg_DateEntered, mtg_DateLastEdited, mtg_EnteredBy, mtg_EditedBy

Types organisateur (MeetingService) :
- family:{fam_ID}  → membres via PersonQuery::filterByFamId()
- group:{grp_ID}   → membres via Person2group2roleP2g2rQuery::filterByGroupId()
- church:0         → église (sChurchName), pas de liste membres fixe

« Organisations » dans le sélecteur :
- church:0 (nom église depuis SystemConfig sChurchName)
- + groupes dont le type (list_lst id=3) contient org/organisation/organization/ministr

Pas de table organization dédiée dans ChurchCRM core.

Table meeting_attendance_mat :
- mat_MeetingId, mat_PersonId, mat_IsPresent (1=présent, 0=absent)
- UNIQUE (meeting, person), FK CASCADE

CORRECTIFS POST-IMPLÉMENTATION

## Bug 500 — Nouvelle réunion
Erreur : « Nom de configuration invalide demandé: sDateTimePickerFormat »
Cause : clé inexistante passée depuis meetings/routes/editor.php
Fix : suppression des appels SystemConfig::getValue('sDateTimePickerFormat')
      (inutilisés ; le formulaire utilise <input type="datetime-local">)

Fichier : src/meetings/routes/editor.php

## UX — Liste présence selon organisateur
Demande : assemblées de maison (groupes) → seuls les membres du groupe dans la liste.

Fix dans src/meetings/views/editor.php (JavaScript) :
- Au changement du sélecteur Organizer → loadMembersFromOrganizer() automatique
- Remplace le tableau présences par les membres du groupe/famille (API /meetings/members)
- Sélecteur « Ajouter une personne » limité aux membres de l'organisateur
- Bouton « Charger les membres de l'organisateur » supprimé
- church:0 → message explicatif, pas de liste globale
- Édition réunion existante : conserve les présences chargées ; filtre le picker add

Prérequis données : le groupe (ex. « Assemblée chez Ghislain ») doit avoir des
membres assignés dans ChurchCRM (Groupes → fiche groupe). Valeur organisateur = group:{grp_ID}
même si affiché dans optgroup « Organizations ».
